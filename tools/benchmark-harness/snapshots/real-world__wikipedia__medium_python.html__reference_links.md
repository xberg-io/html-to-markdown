**Python** is a [high-level][224], [general-purpose programming language][225]. Its design philosophy emphasizes [code readability][226] with the use of [significant indentation][227].[[34]][228]

Python is [dynamically type-checked][229] and [garbage-collected][230]. It supports multiple [programming paradigms][121], including [structured][128] (particularly [procedural][125]), [object-oriented][123] and [functional programming][127].

[Guido van Rossum][131] began working on Python in the late 1980s as a successor to the [ABC][169] programming language. Python 3.0, released in 2008, was a major revision
 and not completely [backward-compatible][231] with earlier versions. Recent versions, such as Python 3.13, 3.12 and older (and
 3.14), have added capabilities and keywords for typing (and more; e.g.
 increasing speed); helping with (optional) [static typing][232].[[35]][233] Currently only versions in the 3.x series are supported.

Python consistently ranks as one of the most popular programming languages, and
 it has gained widespread use in the [machine learning][234] community.[[36]][235][[37]][236][[38]][237][[39]][238] It is widely taught as an introductory programming language.[[40]][239]

### History

[[edit][240]]

Main article: [History of Python][241]

[/wiki/File:Guido_van_Rossum_in_PyConUS24.jpg][243]

*The designer of Python, [Guido van Rossum][131], at PyCon US 2024*

Python was conceived in the late 1980s[[41]][244] by [Guido van Rossum][131] at [Centrum Wiskunde & Informatica][245] (CWI) in the [Netherlands][246].[[42]][247] It was designed as a successor to the [ABC][169] programming language, which was inspired by [SETL][248],[[43]][249] capable of [exception handling][250] and interfacing with the [Amoeba][251] operating system.[[13]][170] Python implementation began in December, 1989.[[44]][252] Van Rossum first released it in 1991 as Python 0.9.0.[[42]][247] Van Rossum assumed sole responsibility for the project, as the lead developer,
 until 12 July 2018, when he announced his "permanent vacation" from
 responsibilities as Python's "[benevolent dictator for life][253]" (BDFL); this title was bestowed on him by the Python community to reflect his
 long-term commitment as the project's chief decision-maker.[[45]][254] (He has since come out of retirement and is self-titled "BDFL-emeritus".) In
 January 2019, active Python core developers elected a five-member Steering
 Council to lead the project.[[46]][255][[47]][256]

The name *Python* derives from the British comedy series [Monty Python's Flying Circus][257].[[48]][258] (See [§ Naming][259].)

Python 2.0 was released on 16 October 2000, with many major new features such as [list comprehensions][260], [cycle-detecting][261] garbage collection, [reference counting][262], and [Unicode][263] support.[[49]][264] Python 2.7's [end-of-life][265] was initially set for 2015, and then postponed to 2020 out of concern that a
 large body of existing code could not easily be forward-ported to
 Python 3.[[50]][266][[51]][267] It no longer receives security patches or updates.[[52]][268][[53]][269] While Python 2.7 and older versions are officially unsupported, a different
 unofficial Python implementation, [PyPy][158],
 continues to support Python 2, i.e., "2.7.18+" (plus 3.10), with the plus
 signifying (at least some) "[backported][270] security updates".[[54]][271]

Python 3.0 was released on 3 December 2008, and was a major revision and
 not completely [backward-compatible][231] with earlier versions, with some new semantics and changed syntax.
 Python 2.7.18, released in 2020, was the last release of Python 2.[[55]][272] Several releases in the Python 3.x series have added new syntax to the language,
 and made a few (considered very minor) backwards-incompatible changes.

As of 14 August 2025[[update]][273], Python 3.13 is the latest stable release and Python 3.10 is the oldest
 supported release.[[56]][274] Releases receive two years of full support followed by three years of security
 support.

### Design philosophy and features

[[edit][275]]

Python is a [multi-paradigm programming language][276]. Object-oriented programming and structured programming are fully supported,
 and many of their features support functional programming and [aspect-oriented programming][277] – including [metaprogramming][278][[57]][279] and [metaobjects][280].[[58]][281] Many other paradigms are supported via extensions, including [design by contract][282][[59]][283][[60]][284] and [logic programming][285].[[61]][286] Python is often referred to as a *['glue language'][287]*[[62]][288] because it is purposely designed to be able to integrate components written in
 other languages.

Python uses dynamic typing and a combination of [reference counting][262] and a cycle-detecting garbage collector for [memory management][289].[[63]][290] It uses dynamic [name resolution][291] ([late binding][292]), which
 binds method and variable names during program execution.

Python's design offers some support for functional programming in the "[Lisp][190] tradition". It has `filter`, `map`, and `reduce` functions; [list comprehensions][260], [dictionaries][293], [sets][294], and [generator][295] expressions.[[64]][296] The standard library has two modules (`itertools` and `functools`) that implement functional tools borrowed from [Haskell][186] and [Standard ML][195].[[65]][297]

Python's core philosophy is summarized in the [Zen of Python][298] (PEP 20)
 written by [Tim Peters][299], which includes aphorisms such as these:[[66]][300]

- Explicit is better than implicit.
- Simple is better than complex.
- Readability counts.
- Special cases aren't special enough to break the rules.
- Although practicality beats purity.
- Errors should never pass silently.
- Unless explicitly silenced.
- There should be one-- and preferably only one --obvious way to do it.


However, Python have received criticism for violating these principles and
 adding unnecessary language bloat.[[67]][301] Responses to these criticisms note that the Zen of Python is a guideline rather
 than a rule.[[68]][302] The addition of some new features had been controversial: Guido van Rossum
 resigned as *Benevolent Dictator for Life* after conflict about adding the
 assignment expression operator in Python 3.8 .[[69]][303][[70]][304]

Nevertheless, rather than building all functionality into its core, Python was
 designed to be highly [extensible][305] via modules. This compact modularity has made it particularly popular as a means
 of adding programmable interfaces to existing applications. Van Rossum's vision
 of a small core language with a large standard library and easily extensible
 interpreter stemmed from his frustrations with ABC, which represented the
 opposite approach.[[41]][244]

Python claims to strive for a simpler, less-cluttered syntax and grammar, while
 giving developers a choice in their coding methodology. In contrast to [Perl][193]'s motto "[there is more than one way to do it][306]", Python advocates an approach where "there should be one – and preferably
 only one – obvious way to do it".[[66]][300] In practice, however, Python provides many ways to achieve a given goal. There
 are, for example, at least three ways to format a string literal, with no
 certainty as to which one a programmer should use.[[71]][307] [Alex Martelli][308] is a [Fellow][309] at the [Python Software Foundation][133] and Python book author; he wrote that "To describe something as 'clever' is *not* considered a compliment in the Python culture."[[72]][310]

Python's developers usually try to avoid [premature optimization][311]; they also reject patches to non-critical parts of the [CPython][157] reference implementation
 that would offer marginal increases in speed at the cost of clarity.[[73]][312][*[failed verification][173]*] Execution speed can be improved by moving speed-critical functions to extension
 modules written in languages such as [C][178],
 or by using a [just-in-time compiler][313] like [PyPy][158]. It is also possible to [cross-compile to other languages][314]; but this approach either fails to achieve the expected speed-up, since Python
 is a very [dynamic language][315], or only a restricted subset of Python is compiled (with potential minor
 semantic changes).[[74]][316]

Python's developers aim for the language to be fun to use. This goal is
 reflected in the name – a tribute to the British comedy group [Monty Python][317][[75]][318] – and in playful approaches to some tutorials and reference materials. For
 instance, some code examples use the terms "spam" and "eggs" (in reference to [a Monty Python sketch][319]), rather than the typical terms ["foo" and "bar"][320].[[76]][321][[77]][322]

A common [neologism][323] in the Python
 community is *pythonic*, which has a wide range of meanings related to
 program style: Pythonic code may use Python [idioms][324] well; be
 natural or show fluency in the language; or conform with Python's minimalist
 philosophy and emphasis on readability.[[78]][325]

### Syntax and semantics

[[edit][326]]

Main article: [Python syntax and semantics][327]

[/wiki/File:Hello_World_in_Python.png][329]

*An example of Python code and indentation*

[/wiki/File:Af-Helloworld_(C_Sharp).svg][331]

*Example of [C#][332] code with curly braces and semicolons*

Python is meant to be an easily readable language. Its formatting is visually
 uncluttered and often uses English keywords where other languages use
 punctuation. Unlike many other languages, it does not use [curly brackets][333] to delimit blocks, and semicolons after statements are allowed but rarely used.
 It has fewer syntactic exceptions and special cases than [C][178] or [Pascal][334].[[79]][335]

#### Indentation

[[edit][336]]

Further information: [Python syntax and semantics § Indentation][337]

Python uses [whitespace][338] indentation, rather than curly brackets or keywords, to delimit [blocks][339]. An
 increase in indentation comes after certain statements; a decrease in
 indentation signifies the end of the current block.[[80]][340] Thus, the program's visual structure accurately represents its semantic
 structure.[[81]][341] This feature is sometimes termed the [off-side rule][342]. Some
 other languages use indentation this way; but in most, indentation has no
 semantic meaning. The recommended indent size is four spaces.[[82]][343]

#### Statements and control flow

[[edit][344]]

Python's [statements][345] include the following:

- The [assignment][346] statement, using a single equals sign `=`
- The `[if][347]` statement, which conditionally executes a block of code, along with `[else][348]` and `elif` (a contraction of `[else if][349]`)
- The `[for][350]` statement, which iterates over an *iterable* object, capturing each

  element to a local variable for use by the attached block

- The `[while][351]` statement, which executes a block of code as long as boolean condition is true
- The `[try][352]` statement, which allows exceptions raised in its attached code block to be

  caught and handled by `except` clauses (or new syntax `except*` in Python 3.11 for exception groups[[83]][353]); the `try` statement also ensures that clean-up code in a `finally` block is always run regardless of how the block exits

- The `raise` statement, used to raise a specified exception or

  re-raise a caught exception

- The `class` statement, which executes a block of code and attaches

  its local namespace to a [class][354], for use in object-oriented programming

- The `def` statement, which defines a [function][355] or [method][356]
- The `[with][357]` statement, which encloses a code block within a context manager, allowing [resource-acquisition-is-initialization][358] (RAII)-like behavior and replacing a common try/finally idiom[[84]][359] Examples of a context include acquiring a [lock][360] before some code is run, and then releasing the lock; or opening and then

  closing a [file][361]

- The `[break][362]` statement, which exits a loop
- The `continue` statement, which skips the rest of the current

  iteration and continues with the next

- The `del` statement, which removes a variable—deleting the

  reference from the name to the value, and producing an error if the variable
  is referred to before it is redefined [[c]][363]

- The `pass` statement, serving as a [NOP][364] (i.e., no operation),

  which is syntactically needed to create an empty code block

- The `[assert][365]` statement, used in debugging to check for conditions that should apply
- The `yield` statement, which returns a value from a [generator][366] function (and also an operator); used to implement [coroutines][367]
- The `return` statement, used to return a value from a function
- The `[import][368]` and `from` statements, used to import modules whose functions or

  variables can be used in the current program

- The `match` and `case` statements, analogous to a [switch statement][369] construct, which compares an expression against one or more cases as a

  control-flow measure


The assignment statement (`=`) binds a name as a [reference][370] to a separate, dynamically allocated [object][371]. Variables may subsequently be rebound at any time to any object. In Python, a
 variable name is a generic reference holder without a fixed [data type][140]; however, it
 always refers to *some* object with a type. This is called [dynamic typing][372]—in contrast to [statically-typed][373] languages, where each variable may contain only a value of a certain type.

Python does not support [tail call][374] optimization or [first-class continuations][375]; according to Van Rossum, the language never will.[[85]][376][[86]][377] However, better support for [coroutine][367]-like functionality is
 provided by extending Python's generators.[[87]][378] Before 2.5, generators were [lazy][379] [iterators][380]; data was passed
 unidirectionally out of the generator. From Python 2.5 on, it is possible
 to pass data back into a generator function; and from version 3.3, data can be
 passed through multiple stack levels.[[88]][381]

#### Expressions

[[edit][382]]

Python's [expressions][383] include the following:

- The `+`, `-`, and `*` operators for

  mathematical addition, subtraction, and multiplication are similar to other
  languages, but the behavior of division differs. There are two types of
  division in Python: [floor division][384] (or integer division) `//`, and floating-point division `/`.[[89]][385] Python uses the `**` operator for exponentiation.

- Python uses the `+` operator for string concatenation. The language

  uses the `*` operator for duplicating a string a specified number
  of times.

- The `@` infix operator is intended to be used by libraries such as [NumPy][386] for [matrix multiplication][387].[[90]][388][[91]][389]
- The syntax `:=`, called the " walrus operator", was introduced in Python 3.8. This operator assigns values to variables as

  part of a larger expression.[[92]][390]

- In Python, `==` compares two objects by value. Python's `is` operator may be used to compare object identities (i.e.,

  comparison by reference), and comparisons may be chained—for example, `a <=
  b <=
  c`.

- Python uses `and`, `or`, and `not` as Boolean

  operators.

- Python has a type of expression called a *[list comprehension][391]*, and a more general expression called a *generator expression*.[[64]][296]
- [Anonymous functions][392] are implemented using [lambda expressions][393]; however, there may be only one expression in each body.
- Conditional expressions are written as `x if c
  else y`.[[93]][394] (This is different in operand order from the `[c ? x : y][395]` operator common to many other languages.)
- Python makes a distinction between [lists][396] and [tuples][397]. Lists are written as `[1,
  2, 3]`, are mutable, and cannot be used as the keys of dictionaries (since

  dictionary keys must be [immutable][398] in Python). Tuples, written as `(1,
  2, 3)`, are immutable and thus can be used as the keys of dictionaries, provided
  that all of the tuple's elements are immutable. The `+` operator
  can be used to concatenate two tuples, which does not directly modify their
  contents, but produces a new tuple containing the elements of both. For
  example, given the variable `t` initially equal to `(1,
  2, 3)`, executing `t = t
                        + (4, 5)` first evaluates `t + (4, 5)`, which yields `(1,
  2, 3, 4,
  5)`; this result is then assigned back to `t`—thereby effectively
  "modifying the contents" of `t` while conforming to the immutable
  nature of tuple objects. Parentheses are optional for tuples in unambiguous
  contexts.[[94]][399]

- Python features *sequence unpacking* where multiple expressions, each

  evaluating to something assignable (e.g., a variable or a writable property)
  are associated just as in forming tuple literal; as a whole, the results are
  then put on the left-hand side of the equal sign in an assignment statement.
  This statement expects an *iterable* object on the right-hand side of the
  equal sign to produce the same number of values as the writable expressions on
  the left-hand side; while iterating, the statement assigns each of the values
  produced on the right to the corresponding expression on the left.[[95]][400]

- Python has a "string format" operator `%` that functions

  analogously to `[printf][401]` format strings
  in the C language—e.g. `"spam=%s eggs=%d" %
  ("blah", 2)` evaluates to `"spam=blah eggs=2"`. In Python 2.6+ and 3+, this
  operator was supplemented by the `format()` method of the `str` class, e.g., `"spam={0} eggs={1}".format("blah",
  2)`. Python 3.6 added "f-strings": `spam =
  "blah";
  eggs =
  2; f'spam={spam} eggs={eggs}'`.[[96]][402]

- Strings in Python can be [concatenated][403] by "adding" them (using the same operator as for adding integers and floats);

  e.g., `"spam" +
  "eggs"` returns `"spameggs"`. If strings contain numbers, they are
  concatenated as strings rather than as integers, e.g. `"2" +
  "2"` returns `"22"`.

- Python supports [string literals][404] in

  several ways:

- Delimited by single or double quotation marks; single and double quotation

  marks have equivalent functionality (unlike in [Unix shells][405], [Perl][193], and Perl-influenced
  languages). Both marks use the backslash (`\`) as an [escape character][406]. [String interpolation][407] became available in Python 3.6 as "formatted string literals".[[96]][402]

- Triple-quoted, i.e., starting and ending with three single or double

  quotation marks; this may span multiple lines and function like [here documents][408] in
  shells, Perl, and [Ruby][215].

- [Raw string][409] varieties, denoted by prefixing the string literal with `r`.

  Escape sequences are not interpreted; hence raw strings are useful where
  literal backslashes are common, such as in [regular expressions][410] and [Windows][411]-style paths. (Compare "`@`-quoting" in [C#][332].)

- Python has [array index][412] and [array slicing][413] expressions in lists, which are written as `a[key]`, `a[start:stop]` or `a[start:stop:step]`. Indexes are [zero-based][414], and negative indexes are relative to the end. Slices take elements from the *start* index up to, but not including, the *stop* index. The

  (optional) third slice [parameter][415], called *step* or *stride*, allows elements to be skipped or
  reversed. Slice indexes may be omitted—for example, `a[:]` returns a copy of the entire list. Each element of a slice is a [shallow copy][416].


In Python, a distinction between expressions and statements is rigidly enforced,
 in contrast to languages such as [Common Lisp][417], [Scheme][418], or [Ruby][215]. This distinction leads to duplicating some functionality, for example:

- [List comprehensions][419] vs. `for`-loops
- [Conditional][420] expressions vs. `if` blocks
- The `eval()` vs. `exec()` built-in functions (in

  Python 2, `exec` is a statement); the former function is for
  expressions, while the latter is for statements


A statement cannot be part of an expression; because of this restriction,
 expressions such as list and `dict` comprehensions (and lambda
 expressions) cannot contain statements. As a particular case, an assignment
 statement such as `a =
                      1` cannot be part of the conditional expression of a conditional statement.

#### Methods

[[edit][421]]

[Methods][422] of objects are functions attached to the object's class; the syntax for normal
 methods and functions, `instance.method(argument)`, is [syntactic sugar][423] for `Class.method(instance,
                      argument)`. Python methods have an explicit `[self][424]` parameter to access [instance data][425], in contrast to the implicit self (or `this`) parameter in some
 object-oriented programming languages (e.g., [C++][180], [Java][426], [Objective-C][427], [Ruby][215]).[[97]][428] Python also provides methods, often called *dunder methods* (because their
 names begin and end with double underscores); these methods allow user-defined
 classes to modify how they are handled by native operations including length,
 comparison, [arithmetic][429], and type conversion.[[98]][430]

#### Typing

[[edit][431]]

[/wiki/File:Python_3.13_Standrd_Type_Hierarchy-en.svg][433]

*The standard type hierarchy in Python 3*

Python uses [duck typing][141], and
 it has typed objects but untyped variable names. Type constraints are not
 checked at definition time; rather, operations on an object may fail at usage
 time, indicating that the object is not of an appropriate type. Despite being [dynamically typed][434], Python is [strongly typed][435], forbidding operations that are poorly defined (e.g., adding a number and a
 string) rather than quietly attempting to interpret them.

Python allows programmers to define their own types using [classes][354], most often for [object-oriented programming][123]. New [instances][371] of classes are constructed by calling the class, for example, `SpamClass()` or `EggsClass()`); the classes are instances of the [metaclass][436] `type` (which is an instance of itself), thereby allowing
 metaprogramming and [reflection][129].

Before version 3.0, Python had two kinds of classes, both using the same syntax: *old-style* and *new-style*.[[99]][437] Current Python versions support the semantics of only the new style.

Python supports [optional type annotations][145].[[5]][438][[100]][439] These annotations are not enforced by the language, but may be used by external
 tools such as **mypy** to catch errors. Python includes a module `typing` including several type names for type annotations.[[101]][440][[102]][441] Mypy also supports a Python compiler called mypyc, which leverages type

#### Arithmetic operations

[[edit][459]]

Python includes conventional symbols for arithmetic operators (`+`, `-`, `*`, `/`), the floor-division operator `//`, and the [modulo operator][460] `%`. (With the modulo operator, a remainder can be negative, e.g., `4 % -3 == -2`.) Python also offers the `**` symbol
 for [exponentiation][461],
 e.g. `5**3 == 125` and `9**0.5 == 3.0`; it also offers the
 matrix‑multiplication operator `@` .[[107]][462] These operators work as in traditional mathematics; with the same [precedence rules][463], the [infix][464] operators `+` and `-` can also be [unary][465], to represent
 positive and negative numbers respectively.

Division between integers produces floating-point results. The behavior of
 division has changed significantly over time:[[108]][466]

- The current version of Python (i.e., since 3.0) changed the `/` operator to always represent floating-point division, e.g., `5/2
  == 2.5`.
- The floor division `//` operator was introduced, meaning that `7//3 == 2`, `-7//3 == -3`, `7.5//3 == 2.0`,

  and `-7.5//3 == -3.0`. For Python 2.7, adding the `from __future__ import
  division` statement allows a module in Python 2.7 to use Python 3.x rules for division
  (see above).


In Python terms, the `/` operator represents *true division* (or
 simply *division*), while the `//` operator represents *floor division.* Before version 3.0, the `/` operator
 represents *classic division*.[[108]][466]

[Rounding][467] towards negative
 infinity, though a different method than in most languages, adds consistency to
 Python. For instance, this rounding implies that the equation `(a +
                      b)//b == a//b +
                      1` is always true. The rounding also implies that the equation `b*(a//b) + a%b ==
                      a` is valid for both positive and negative values of `a`. As expected,
 the result of `a%b` lies in the [half-open interval][468] [0, *b*), where `b` is a positive integer; however, maintaining
 the validity of the equation requires that the result must lie in the interval
 (*b*, 0] when `b` is negative.[[109]][469]

Python provides a `round` function for rounding a float to the
 nearest integer. For [tie-breaking][470],
 Python 3 uses the *round to even* method: `round(1.5)` and `round(2.5)` both produce `2`.[[110]][471] Python versions before 3 used the [round-away-from-zero][472] method: `round(0.5)` is `1.0`, and `round(-0.5)` is `−1.0`.[[111]][473]

Python allows Boolean expressions that contain multiple equality relations to be
 consistent with general usage in mathematics. For example, the expression `a < b < c` tests whether `a` is less than `b` and `b` is less than `c`.[[112]][474] C-derived languages interpret this expression differently: in C, the expression
 would first evaluate `a < b`, resulting in 0 or 1, and that result
 would then be compared with `c`.[[113]][475]

Python uses [arbitrary-precision arithmetic][476] for all integer operations. The `Decimal` type/class in the `decimal` module provides [decimal floating-point numbers][477] to a pre-defined arbitrary precision with several rounding modes.[[114]][478] The `Fraction` class in the `fractions` module provides
 arbitrary precision for [rational numbers][479].[[115]][480]

Due to Python's extensive mathematics library and the third-party library [NumPy][386], the language is frequently used
 for scientific scripting in tasks such as numerical data processing and
 manipulation.[[116]][481][[117]][482]

#### Function syntax

[[edit][483]]

[Functions][484] are created in Python by using the `def` keyword. A function is
 defined similarly to how it is called, by first providing the function name and
 then the required parameters. Here is an example of a function that prints its
 inputs:

```text
def printer(input1: str, input2: str = "already there") -> None:
    print(input1)
    print(input2)

printer("hello")

# Example output:
# hello
# already there
```

To assign a default value to a function parameter in case no actual value is
 provided at run time, variable-definition syntax can be used inside the function
 header.

### Implementations

[[edit][538]]

See also: [List of Python software § Python implementations][539]

#### Reference implementation

[[edit][540]]

[CPython][157] is the [reference implementation][541] of Python. This implementation is written in C, meeting the [C11][542] standard[[127]][543] since version 3.11. Older versions use the [C89][544] standard with several select [C99][545] features,
 but third-party extensions are not limited to older C versions—e.g., they can be
 implemented using C11 or C++.[[128]][546][[129]][547] CPython [compiles][548] Python programs
 into an intermediate [bytecode][549],[[130]][550] which is then executed by a [virtual machine][551].[[131]][552] CPython is distributed with a large standard library written in a combination of
 C and native Python.

CPython is available for many platforms, including Windows and most modern [Unix-like][553] systems, including
 macOS (and [Apple M1][554] Macs, since
 Python 3.9.1, using an experimental installer). Starting with
 Python 3.9, the Python installer intentionally fails to install on [Windows 7][555] and 8;[[132]][556][[133]][557] [Windows XP][558] was supported until
 Python 3.5, with unofficial support for [VMS][559].[[134]][560] Platform portability was one of Python's earliest priorities.[[135]][561] During development of Python 1 and 2, even [OS/2][562] and [Solaris][563] were supported;[[136]][564] since that time, support has been dropped for many platforms.

All current Python versions (since 3.7) support only operating systems that
 feature multithreading, by now supporting not nearly as many operating systems
 (dropping many outdated) than in the past.

#### Other implementations

[[edit][565]]

All alternative implementations have at least slightly different semantics. For
 example, an alternative may include unordered dictionaries, in contrast to other
 current Python versions. As another example in the larger Python ecosystem, PyPy
 does not support the full C Python API. Alternative implementations include the
 following:

- [PyPy][158] is a fast, compliant interpreter of

  Python 2.7 and 3.10.[[137]][566][[138]][567] PyPy's [just-in-time compiler][313] often improves speed significantly relative to CPython, but PyPy does not
  support some libraries written in C.[[139]][568] PyPy offers support for the [RISC-V][569] instruction-set architecture.

- Codon is an implementation with an [ahead-of-time (AOT) compiler][570], which compiles a statically-typed Python-like language whose "syntax and

  semantics are nearly identical to Python's, there are some notable
  differences"[[140]][571] For example, Codon uses 64-bit machine integers for speed, not arbitrarily as
  with Python; Codon developers claim that speedups over CPython are usually on
  the order of ten to a hundred times. Codon compiles to machine code (via [LLVM][572]) and supports native
  multithreading.[[141]][573] Codon can also compile to Python extension modules that can be imported and
  used from Python.

- [MicroPython][159] and [CircuitPython][160] are

  Python 3 variants that are optimized for [microcontrollers][574],
  including the [Lego Mindstorms EV3][575].[[142]][576]

- Pyston is a variant of the Python runtime that uses just-in-time compilation

  to speed up execution of Python programs.[[143]][577]

- Cinder is a performance-oriented fork of CPython 3.8 that features a number of

  optimizations, including bytecode inline caching, eager evaluation of
  coroutines, a method-at-a-time [JIT][578], and an experimental bytecode compiler.[[144]][579]

- The Snek[[145]][580][[146]][581][[147]][582] embedded computing language "is Python-inspired, but it is not Python. It is

  possible to write Snek programs that run under a full Python system, but most
  Python programs will not run under Snek."[[148]][583] Snek is compatible with 8-bit [AVR microcontrollers][584] such as [ATmega 328P][585]-based Arduino, as well as larger microcontrollers that are compatible with [MicroPython][159]. Snek is an
  imperative language that (unlike Python) omits [object-oriented programming][123]. Snek supports only one numeric data type, which features 32-bit [single precision][586] (resembling [JavaScript][205] numbers, though
  smaller).


#### Unsupported implementations

[[edit][587]]

[Stackless Python][163] is a significant fork of CPython that implements [microthreads][588]. This
 implementation uses the [call stack][589] differently, thus
 allowing massively concurrent programs. PyPy also offers a stackless
 version.[[149]][590]

Just-in-time Python compilers have been developed, but are now unsupported:

- Google began a project named [Unladen Swallow][591] in

  2009: this project aimed to speed up the Python interpreter five-fold by using [LLVM][572], and improve [multithreading][592] capability for scaling to thousands of cores,[[150]][593] while typical implementations are limited by the [global interpreter lock][594].

- [Psyco][595] is a discontinued [just-in-time][578] [specializing][596] compiler, which integrates with CPython and transforms bytecode to machine

  code at runtime. The emitted code is specialized for certain [data types][597] and is faster than
  standard Python code. Psyco does not support Python 2.7 or later.

- [PyS60][598] was a

  Python 2 interpreter for [Series 60][599] mobile phones, which was released by [Nokia][600] in 2005. The interpreter
  implemented many modules from Python's standard library, as well as additional
  modules for integration with the [Symbian][601] operating system. The
  Nokia [N900][602] also
  supports Python through the [GTK][603] widget
  library, allowing programs to be written and run on the target device.[[151]][604]


#### Cross-compilers to other languages

[[edit][605]]

There are several compilers/[transpilers][606] to high-level object languages; the source language is unrestricted Python, a
 subset of Python, or a language similar to Python:

- Brython[[152]][607] and Transcrypt[[153]][608][[154]][609] compile Python to [JavaScript][205].
- [Cython][165] compiles a superset of Python

  to C. The resulting code can be used with Python via direct C-level API calls
  into the Python interpreter.

- PyJL compiles/transpiles a subset of Python to "human-readable, maintainable,

  and high-performance Julia source code".[[74]][316] Despite the developers' performance claims, this is not possible for *arbitrary* Python code; that is, compiling to a faster language or
  machine code is known to be impossible in the general case. The semantics of
  Python might potentially be changed, but in many cases speedup is possible
  with few or no changes in the Python code. The faster Julia source code can
  then be used from Python or compiled to machine code.

- [Nuitka][610] compiles Python into C.[[155]][611] This compiler works with Python 3.4 to 3.12 (and 2.6 and 2.7) for Python's

  main supported platforms (and Windows 7 or even Windows XP) and for Android.
  The compiler developers claim full support for Python 3.10, partial support
  for Python 3.11 and 3.12, and experimental support for Python 3.13. Nuitka
  supports macOS including Apple Silicon-based versions. The compiler is free of
  cost, though it has commercial add-ons (e.g., for hiding source code).

- [Numba][612] is a JIT compiler that is used

  from Python; the compiler translates a subset of Python and NumPy code into
  fast machine code. This tool is enabled by adding a decorator to the relevant
  Python code.

- Pythran compiles a subset of Python 3 to C++ ([C++11][613]).[[156]][614]
- [RPython][166] can be

  compiled to C, and it is used to build the PyPy interpreter for Python.

- The Python → 11l → C++ transpiler[[157]][615] compiles a subset of Python 3 to C++ ([C++17][616]).


There are also specialized compilers:

- [MyHDL][617] is a Python-based [hardware description language][618] (HDL) that converts MyHDL code to [Verilog][619] or [VHDL][620] code.


Some older projects existed, as well as compilers not designed for use with
 Python 3.x and related syntax:

- Google's Grumpy [transpiles][621] Python 2 to [Go][204].[[158]][622][[159]][623][[160]][624] The latest release was in 2017.
- [IronPython][161] allows running

  Python 2.7 programs with the .NET [Common Language Runtime][625].[[161]][626] An [alpha][627] version (released in 2021), is available for "Python 3.4, although
  features and behaviors from later versions may be included."[[162]][628]

- [Jython][162] compiles Python 2.7 to

  Java bytecode, allowing the use of Java libraries from a Python program.[[163]][629]

- [Pyrex][630] (last released in 2010) and [Shed Skin][631] (last released in

2013) compile to C and C++ respectively.


#### Performance

[[edit][632]]

A performance comparison among various Python implementations, using a
 non-numerical (combinatorial) workload, was presented at EuroSciPy '13.[[164]][633] In addition, Python's performance relative to other programming languages is
 benchmarked by [The Computer Language Benchmarks Game][634].[[165]][635]

There are several approaches to optimizing Python performance, given the
 inherent slowness of an [interpreted language][636]. These approaches include the following strategies or tools:

- [Just-in-time compilation][578]: Dynamically compiling Python code just before it is executed. This

  technique is used in libraries such as [Numba][612] and [PyPy][158].

- [Static compilation][548]: Python code

  is compiled into machine code sometime before execution. An example of this
  approach is Cython, which compiles Python into C.

- Concurrency and parallelism: Multiple tasks can be run simultaneously. Python

  contains modules such as `multiprocessing` to support this form of
  parallelism. Moreover, this approach helps to overcome limitations of the [Global Interpreter Lock][594] (GIL) in CPU tasks.

- Efficient data structures: Performance can also be improved by using data

  types such as `Set` for membership tests, or `deque` from `collections` for [queue][637] operations.


### Language Development

[[edit][638]]

Python's development is conducted largely through the *Python Enhancement Proposal* (PEP) process; this process is the primary
 mechanism for proposing major new features, collecting community input on
 issues, and documenting Python design decisions.[[166]][639] Python coding style is covered in PEP 8.[[167]][640] Outstanding PEPs are reviewed and commented on by the Python community and the
 steering council.[[166]][639]

Enhancement of the language corresponds with development of the CPython
 reference implementation. The mailing list python-dev is the primary forum for
 the language's development. Specific issues were originally discussed in the [Roundup][641] [bug tracker][642] hosted by the foundation.[[168]][643] In 2022, all issues and discussions were migrated to [GitHub][644].[[169]][645] Development originally took place on a [self-hosted][646] source-code repository running [Mercurial][647], until Python moved to [GitHub][644] in January 2017.[[170]][648]

CPython's public releases have three types, distinguished by which part of the
 version number is incremented:

- *Backward-incompatible versions*, where code is expected to break and

  must be manually [ported][649]. The
  first part of the version number is incremented. These releases happen
  infrequently—version 3.0 was released 8 years after 2.0. According to Guido
  van Rossum, a version 4.0 will probably never exist.[[171]][650]

- *Major or "feature" releases* are largely compatible with the previous

  version but introduce new features. The second part of the version number is
  incremented. Starting with Python 3.9, these releases are expected to
  occur annually.[[172]][651][[173]][652] Each major version is supported by bug fixes for several years after its
  release.[[174]][653]

- *Bug fix releases*,[[175]][654] which introduce no new features, occur approximately every three months; these

  releases are made when a sufficient number of bugs have been fixed [upstream][655] since the last release. Security vulnerabilities are also patched in these
  releases. The third and final part of the version number is incremented.[[175]][654]


Many [alpha, beta, and release-candidates][656] are also released as previews and for testing before final releases. Although
 there is a rough schedule for releases, they are often delayed if the code is
 not ready yet. Python's development team monitors the state of the code by
 running a large [unit test][657] suite during development.[[176]][658]

The major [academic conference][659] on Python is [PyCon][660].
 There are also special Python mentoring programs, such as [PyLadies][661].

- [PyQt][681] and [PyGTK][682], which bind [Qt][683] and GTK to Python

  respectively;

- [PyPy][158], a Python implementation originally

  written in Python;

- [NumPy][386], a Python library for numerical

  processing.

- [Jupyter][684], a [notebook interface][685] and associated project for interactive computing


### Popularity

[[edit][686]]

Since 2003, Python has consistently ranked in the top ten of the most popular
 programming languages in the [TIOBE Programming Community Index][687]; as of December 2022[[update]][273], Python was the most popular language.[[38]][237] Python was selected as Programming Language of the Year (for "the highest rise
 in ratings in a year") in 2007, 2010, 2018, 2020, 2021, and 2024 —the only
 language to have done so six times as of 2025[[update]][273][[183]][688]). In the TIOBE Index, monthly rankings are based on the volume of searches for
 programming languages on Google, Amazon, Wikipedia, Bing, and 20 other
 platforms. According to the accompanying graph, Python has shown a marked upward
 trend since the early 2000s, eventually passing more established languages such
 as C, C++, and Java. This trend can be attributed to Python's readable syntax,
 comprehensive standard library, and application in data science and machine
 learning fields.[[184]][689]

 languages*

Large organizations that use Python include [Wikipedia][692], [Google][693],[[185]][694] [Yahoo!][695],[[186]][696] [CERN][697],[[187]][698] [NASA][699],[[188]][700] [Facebook][701],[[189]][702] [Amazon][703], [Instagram][704],[[190]][705] [Spotify][706],[[191]][707] and some smaller entities such as [Industrial Light & Magic][708][[192]][709] and [ITA][710].[[193]][711] The social news networking site [Reddit][712] was developed mostly in
 Python.[[194]][713] Organizations that partly use Python include [Discord][714][[195]][715] and [Baidu][716].[[196]][717]

### Types of use

[[edit][718]]

Further information: [List of Python software][719]

[/wiki/File:Python_Powered.png][721]

*Software that is powered by Python*

Python has many uses, including the following:

- [Scripting][722] for [web applications][723]
- Scientific computing
- [Artificial intelligence][724] and [machine learning][234] projects
- [Graphical user interfaces][492] and [desktop environments][725]
- Embedded scripting in software and hardware products
- Operating systems
- [Information security][726]


Python can serve as a scripting language for web applications, e.g., via the [mod_wsgi][727] module for the [Apache web server][728].[[197]][729] With [Web Server Gateway Interface][496], a standard API has evolved to facilitate these applications. [Web frameworks][513] such as [Django][730], [Pylons][731], [Pyramid][732], [TurboGears][733], [web2py][734], [Tornado][735], [Flask][736],
 Bottle, and [Zope][737] support developers in the
 design and maintenance of complex applications. Pyjs and [IronPython][161] can be used to
 develop the client-side of Ajax-based applications. [SQLAlchemy][738] can be used as a [data mapper][739] to a relational database. [Twisted][740] is a
 framework to program communication between computers; this framework is used by [Dropbox][741], for example.

Libraries such as [NumPy][386], [SciPy][742] and [Matplotlib][743] allow the effective
 use of Python in scientific computing,[[198]][744][[199]][745] with specialized libraries such as [Biopython][746] and [Astropy][747] providing domain-specific
 functionality. [SageMath][748] is a [computer algebra system][749] with a [notebook interface][685] that is programmable in Python; the SageMath library covers many aspects of [mathematics][750], including [algebra][751], [combinatorics][752], [numerical mathematics][753], [number theory][754], and [calculus][755].[[200]][756] [OpenCV][757] has Python bindings with a rich
 set of features for [computer vision][758] and [image processing][505].[[201]][759]

Python is commonly used in artificial-intelligence and machine-learning
 projects, with support from libraries such as [TensorFlow][760], [Keras][761], [Pytorch][762], [scikit-learn][763] and [ProbLog][764] (a logic language).[[202]][765][[203]][766][[204]][767][[205]][768][[206]][769] As a scripting language with a [modular architecture][770], simple syntax, and rich text processing tools, Python is often used for [natural language processing][771].[[207]][772]

The combination of Python and [Prolog][773] has proven useful for AI
 applications, with Prolog providing knowledge representation and reasoning
 capabilities. The Janus system, in particular, exploits similarities between
 these two languages, in part because of their dynamic typing and their simple,
 recursive data structures. This combination is typically applied natural
 language processing, visual query answering, geospatial reasoning, and handling
 semantic web data.[[208]][774][[209]][775] The Natlog system, implemented in Python, uses [Definite Clause Grammars][776] (DCGs) to create prompts for two types of generators: text-to-text generators
 such as GPT3, and text-to-image generators such as DALL-E or Stable
 Diffusion.[[210]][777]

Python can be used for graphical user interfaces (GUIs), by using libraries such
 as [Tkinter][778].[[211]][779] Similarly, for the [One Laptop per Child][780] XO computer, most of the [Sugar][781] desktop environment is written in Python (as of 2008).[[212]][782]

Python is embedded in many software products (and some hardware products) as a
 scripting language. These products include the following:

- [finite element method][783] software such as [Abaqus][784],
- 3D parametric modelers such as [FreeCAD][785],
- 3D animation packages such as [3ds Max][786], [Blender][787], [Cinema 4D][788], [Lightwave][789], [Houdini][790], [Maya][791], [modo][792], [MotionBuilder][793], [Softimage][794],
- the visual effects compositor [Nuke][795],
- 2D imaging programs such as [GIMP][796],[[213]][797] [Inkscape][798], [Scribus][799] and [Paint Shop Pro][800],[[214]][801] and
- [musical notation][802] programs such as [scorewriter][803] and [capella][804].


Similarly, [GNU Debugger][805] uses Python
 as a [pretty printer][806] to show complex structures such as C++ containers. [Esri][807] promotes Python as the best choice
 for writing scripts in [ArcGIS][808].[[215]][809] Python has also been used in several video games,[[216]][810][[217]][811] and it has been adopted as first of the three [programming languages][812] available in [Google App Engine][813] (the other two being [Java][814] and [Go][204]).[[218]][815] [LibreOffice][816] includes Python,
 and its developers plan to replace Java with Python; LibreOffice's Python
 Scripting Provider is a core feature[[219]][817] since version 4.0 (from 7 February 2013).

Among hardware products, the [Raspberry Pi][818] [single-board computer][819] project has adopted Python as its main user-programming language.

Many operating systems include Python as a standard component. Python ships with
 most [Linux distributions][820],[[220]][821] [AmigaOS 4][822] (using
 Python 2.7), [FreeBSD][823] (as a
 package), [NetBSD][824], and [OpenBSD][825] (as a package); it can be
 used from the command line (terminal). Many Linux distributions use installers
 written in Python: [Ubuntu][826] uses the [Ubiquity][827] installer, while [Red Hat Linux][828] and [Fedora Linux][829] use the [Anaconda][830] installer. [Gentoo Linux][831] uses Python
 in its [package management system][832], [Portage][833].[[221]][834]

Python is used extensively in the information security industry, including in
 exploit development.[[222]][835][[223]][836]

### Limitations

[[edit][837]]

- The energy usage of Python is much worse than C by a factor of 75.88.[[224]][838]
- Python lacks do while loops.[[225]][839]
- The throughput of Python is worse than C by a factor of 71.9.[[224]][838]
- The average memory usage of Python is worse than C by a factor of 2.4.[[224]][838]
- Creating an executable with Python requires bundling the entire Python

  interpreter into the executable, which causes binary sizes of small executable
  to be massive.[[226]][840]

- Significant whitespace causes Python scripts to be difficult to minify. The

  most compact minification can transform each indent level into a tab.[[227]][841] However, this always takes up at least as much or more than braces.


### Languages influenced by Python

[[edit][842]]

Python's design and philosophy have influenced many other programming languages:

- [Boo][197] uses indentation, a similar syntax, and a similar object model.[[228]][843]
- [Cobra][198] uses indentation and a similar syntax; its *Acknowledgements* document

  lists Python first among influencing languages.[[229]][844]

- [CoffeeScript][199], a

  programming language that cross-compiles to JavaScript, has a Python-inspired
  syntax.

- [ECMAScript][845]–[JavaScript][205] borrowed iterators and [generators][846] from Python.[[230]][847]
- [GDScript][203], a

  Python-like scripting language that is built in to the [Godot][848] game
  engine.[[231]][849]

- [Go][204] is designed for "speed of working in a dynamic language like Python";[[232]][850] Go shares Python's syntax for slicing arrays.
- [Groovy][851] was motivated by a desire to incorporate the Python design philosophy into [Java][426].[[233]][852]
- [Julia][208] was designed to be "as usable for general programming as Python".[[28]][209]
- [Mojo][210] is a non-strict[[29]][211][[234]][853] superset of Python (e.g., omitting classes, and adding [struct][854]).[[235]][855]
- [Nim][212] uses indentation and a similar syntax.[[236]][856]
- [Ruby][215]'s creator, [Yukihiro Matsumoto][857], said that "I wanted a scripting language that was more powerful than Perl,

  and more object-oriented than Python. That's why I decided to design my own
  language."[[237]][858]

- [Swift][217], a programming language developed by Apple, has some Python-inspired

  syntax.[[238]][859]

- [Kotlin][860] blends Python and Java features, which minimizes boilerplate code and enhances

  developer efficiency.[[239]][861]


Python's development practices have also been emulated by other languages. For
 example, Python requires a document that describes the rationale and context for
 any language change; this document is known as a *Python Enhancement Proposal* or PEP. This practice is also used by the
 developers of [Tcl][862],[[240]][863] [Erlang][864],[[241]][865] and Swift.[[242]][866]
